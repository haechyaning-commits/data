# 자체감사결과 스크래핑 재개 방법

`full_scrape.js`는 공공감사포털(pap.go.kr) 자체감사결과 목록을
2025-07-07 ~ 2026-07-07, 기관분류=공공기관 조건으로 전부 내려받는
스크립트입니다. `checkpoint.json`에 진행 상황(다음 페이지, 파일명
중복 처리 registry)이 저장되어 있어 중단 후에도 이어서 실행할 수
있습니다.

## 재개 방법

1. 이 폴더(`_scripts/`)를 작업 디렉터리로 사용할 임시 경로에 복사
   (checkpoint.json은 스크립트와 같은 폴더에 있어야 함)
2. 실행:
   ```
   node full_scrape.js /home/user/data/자체감사결과
   ```
3. `checkpoint.json`의 `nextPage`부터 자동으로 이어서 진행됨
4. 진행되며 생성되는 파일은 `/home/user/data/자체감사결과/`에 저장됨
5. `progress.log`에 전체 처리 로그가 계속 append됨

## 주의

- 프록시 포트가 세션마다 바뀔 수 있으므로 `lib.js`가 `$HTTPS_PROXY`
  환경변수에서 자동으로 읽어옴 (하드코딩 아님)
- 검증 규칙: 각 보고서의 모든 조치사항 첨부파일을 실제로 조회해서
  파일명+크기가 다르면 별도 문서로 취급, 같으면 1개로 합침. 파일명은
  `기관명_{연도}년 {감사분야}` 규칙이며 텍스트가 겹치면 등록일
  최신순으로 `(1)`, `(2)`... 번호를 붙임

## 증분 수집 (최신화) — `incremental_scrape.js`

전량 재수집 없이, 포털에 **새로 등록된 보고서만** 골라 `자체감사결과/`에
이어 붙이는 스크립트입니다. 기존 파일은 건드리지 않는 **append-only** 방식이라
안전합니다. (품질점검 결과, 최초 수집 이후 새로 등록된 보고서가 생길 수 있어
추가했습니다.)

```
node incremental_scrape.js [OUT_DIR] [SINCE] [UNTIL] [--full]
```

- `OUT_DIR` : 저장 폴더 (기본 `/home/user/data/자체감사결과`)
- `SINCE` : 시작일 `YYYYMMDD` (기본 `20250707`)
- `UNTIL` : 종료일 `YYYYMMDD` (기본 오늘 날짜)
- `--full` : 조기 종료 없이 기간 전체를 다시 훑음(정밀 점검용)

**예시** — 최신화만 빠르게:

```
node incremental_scrape.js /home/user/data/자체감사결과
```

**동작**

- 포털 목록은 최신순이라 새 보고서가 앞 페이지에 옵니다. 앞에서부터 훑으며 각
  첨부가 디스크에 "같은 이름·같은 크기"로 이미 있으면 건너뛰고, 없으면 내려받아
  다음 번호로 저장합니다. 연속 2페이지가 통째로 "이미 있음"이면 종료합니다
  (`--full`이면 끝까지).
- 진행 로그는 `incremental.log`에 남습니다. 포털이 파일을 안 내주는 건(HTTP 204)은
  건너뛰고 로그에 기록합니다.

**주의**

- 중복 판정을 `full_scrape.js`의 SHA-256 내용 해시가 아니라 **(이름 + 파일크기)**
  로 합니다. 기존 파일을 재다운로드/재계산하지 않기 위한 것으로, 새 파일 추가에는
  충분합니다. 번호를 처음부터 정밀 재정렬해야 한다면 `full_scrape.js`를 쓰세요.
- 새로 받은 파일을 저장소에 반영하려면 커밋·푸시가 필요합니다(대용량은 배치 권장).
