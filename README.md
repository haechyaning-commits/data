# 공공감사포털 자체감사결과 데이터

[공공감사포털(pap.go.kr)](https://pap.go.kr) 자체감사결과 공개 데이터를 수집한 저장소입니다.

## 폴더 안내

- `자체감사결과/` — 2025-07-07 ~ 2026-07-07 수집분 (아래 "수집 조건" 참고)
- `자체감사파일2/` — 2021-01-01 ~ 2023-12-30 수집분 (아래 "자체감사파일2 (2021~2023 수집분)" 참고)

## 수집 조건

- 시작일: 2025-07-07
- 종료일: 2026-07-07
- 기관분류: 공공기관
- 정렬: 최신순 (API 기본 정렬)
- 총 2,384건의 감사보고서, 첨부파일 10,828개 (10,830개 파일, 그중 1건은 GitHub 100MB 용량 제한으로 2조각 분할)

## 폴더 구조

```
자체감사결과/
  {감사실시기관}_{연도}년 {감사분야}.{확장자}
  {감사실시기관}_{연도}년 {감사분야}(2).{확장자}
  ...
```

- 파일명은 `기관명_연도년 감사분야` 형식입니다 (예: `대한무역투자진흥공사_2026년 종합감사`).
- 감사분야는 API의 `adFldNm` 값(종합감사/특정감사/복무감사/성과감사/재무감사 등)을 사용합니다.
- 같은 기관·연도·감사분야로 여러 건이 등록된 경우, **실제 첨부문서 내용(SHA-256 해시)이 동일하면 번호 없이 1개만 저장**하고, **내용이 다르면 `(1)`, `(2)`... 순서로 번호를 붙여 모두 저장**합니다. 번호는 등록일 기준 최신순으로 매겨집니다.
- 하나의 감사보고서 안에 여러 개의 "처분요구 및 조치사항" 항목이 있는 경우, 각 항목의 첨부파일을 개별적으로 확인하여 실제로 다른 문서인 것만 별도 파일로 저장했습니다 (같은 파일을 공유하는 항목은 중복 저장하지 않음).

## 대용량 파일 안내

`한국동서발전_2026년 특정감사(11).hwpx` 파일은 원본이 GitHub 100MB 업로드 제한을 초과하여 아래 2개로 분할 저장되어 있습니다.

```
한국동서발전_2026년 특정감사(11)_조각(1).hwpx.part
한국동서발전_2026년 특정감사(11)_조각(2).hwpx.part
```

원본으로 복원하려면:

**Windows (명령 프롬프트):**
```
copy /b "한국동서발전_2026년 특정감사(11)_조각(1).hwpx.part"+"한국동서발전_2026년 특정감사(11)_조각(2).hwpx.part" "한국동서발전_2026년 특정감사(11).hwpx"
```

**Mac / Linux:**
```
cat "한국동서발전_2026년 특정감사(11)_조각(1).hwpx.part" "한국동서발전_2026년 특정감사(11)_조각(2).hwpx.part" > "한국동서발전_2026년 특정감사(11).hwpx"
```

## 자체감사파일2 (2021~2023 수집분)

- 시작일: 2021-01-01, 종료일: 2023-12-30
- 기관분류: 공공기관
- 총 9,125건의 감사보고서, 22,406개 첨부문서 (약 7.8GB, 실패 0건)
- 파일명 규칙은 `자체감사결과/`와 동일: `{감사실시기관}_{연도}년 {감사분야}.{확장자}`, 같은 이름이 여러 건이면 `(1)`, `(2)`... 번호 부여 (등록일 최신순). 같은 이름에서 첨부문서의 파일명과 크기까지 동일하면 중복으로 보고 1개만 저장.
- 대용량 분할 파일 1건: `한국도로공사_2022년 특정감사(7).hwp`가 GitHub 100MB 제한을 초과하여 `_조각(1)`, `_조각(2)` 2개의 `.hwp.part`로 분할 저장됨 (복원 방법은 위 "대용량 파일 안내"와 동일).
- 수집 스크립트: `_scripts/build_manifest.js`(메타데이터 수집·파일명 확정) → `_scripts/orchestrate2.sh`(배치 다운로드·커밋·푸시), 전체 파일 목록은 `_scripts/manifest2.json` 참고.

## 다운로드 방법

1. 저장소 최상위에서 브랜치를 `claude/audit-portal-data-organization-7v3j4p`로 전환
2. 초록색 **Code** 버튼 → **Download ZIP**
3. 압축 해제 후 `자체감사결과` 폴더 사용

(GitHub 웹 화면은 폴더당 1,000개 파일까지만 미리보기로 표시하지만, ZIP 다운로드나 `git clone`/`git pull`에는 전체 파일이 포함됩니다.)

## 수집 방식

공공감사포털의 REST API(`/api/fdadPlanRslt`, `/api/files/filelist`, `/api/files/download`)를 직접 호출하여 실제 존재하는 데이터만 수집했습니다. 임의로 생성하거나 추측한 데이터는 없습니다.
